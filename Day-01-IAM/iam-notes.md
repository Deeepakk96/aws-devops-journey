# Day 01 - IAM Roles

## Objective
Learn how IAM Roles work and attach a role to an EC2 instance.

## Hands-on Performed
- Created an IAM Role
- Attached AdministratorAccess policy
- Attached IAM Role to EC2
- Verified Role Permissions

## Screenshots
See the screenshots folder.

## Key Learnings
- IAM Roles provide temporary credentials.
- EC2 should use IAM Roles instead of Access Keys.
- Policies define permissions.
